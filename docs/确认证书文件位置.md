# 确认证书文件位置和修复步骤

## 步骤 1：确认证书文件在主机上的位置

在 FinalShell 终端执行：

```bash
# 检查证书文件是否在主机上
ls -la /home/hsk/ssl/tangxinhao.online/
```

**应该看到**：
```
cert.pem
cert.key
```

如果看不到这两个文件，说明文件没有上传成功，需要重新上传。

---

## 步骤 2：检查 volume 映射

如果文件在主机上存在，但容器内找不到，检查 volume 映射：

```bash
# 检查容器的 volume 映射
docker inspect hsk-nginx | grep -A 30 "Mounts"
```

查看是否有：
```json
{
    "Type": "bind",
    "Source": "/home/hsk/ssl",
    "Destination": "/etc/nginx/ssl",
    ...
}
```

---

## 步骤 3：如果 volume 映射不对，重新创建容器

如果映射不对，需要重新创建：

```bash
cd /home/hsk

# 停止并删除容器
docker stop hsk-nginx
docker rm hsk-nginx

# 重新创建（确保在 /home/hsk 目录下执行）
docker compose -f docker-compose.prod.yml up -d nginx
```

---

## 步骤 4：验证

```bash
# 1. 检查容器内的目录
docker exec hsk-nginx ls -la /etc/nginx/ssl/

# 2. 应该看到 tangxinhao.online 目录
docker exec hsk-nginx ls -la /etc/nginx/ssl/tangxinhao.online/

# 3. 应该看到证书文件
docker exec hsk-nginx ls -la /etc/nginx/ssl/tangxinhao.online/
```
