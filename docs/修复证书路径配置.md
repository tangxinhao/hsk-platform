# 修复证书路径配置

## 问题确认

容器内的 Nginx 配置显示证书路径是：
```
ssl_certificate /etc/letsencrypt/live/tangxinhao.online/fullchain.pem;
ssl_certificate_key /etc/letsencrypt/live/tangxinhao.online/privkey.pem;
```

但实际证书在：
```
/etc/nginx/ssl/tangxinhao.online/cert.pem
/etc/nginx/ssl/tangxinhao.online/cert.key
```

## 解决方案

### 方法一：检查主机上的配置文件

```bash
cat /home/hsk/nginx/conf.d/default.conf | grep -A 5 "listen 443"
```

确认主机上的配置是否正确。

### 方法二：如果主机配置正确，重启容器

```bash
docker restart hsk-nginx
docker logs hsk-nginx --tail 50
```

### 方法三：如果主机配置也不对，修改配置文件

编辑 `/home/hsk/nginx/conf.d/default.conf`，将证书路径改为：
```
ssl_certificate /etc/nginx/ssl/tangxinhao.online/cert.pem;
ssl_certificate_key /etc/nginx/ssl/tangxinhao.online/cert.key;
```

然后重启容器。
