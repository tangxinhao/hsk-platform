# 排查域名解析问题

## 问题确认

`curl -k https://www.tangxinhao.online` 返回：
```
curl: (7) Failed to connect to www.tangxinhao.online port 443 after 387 ms: Couldn't connect to server
```

但端口 443 正在监听，说明可能是域名解析问题。

## 排查步骤

### 步骤 1：检查域名解析

```bash
nslookup www.tangxinhao.online
```

或者：
```bash
dig www.tangxinhao.online
```

确认域名解析到 `118.190.106.159`。

### 步骤 2：直接用 IP 测试（绕过域名解析）

```bash
curl -k https://118.190.106.159 -H "Host: www.tangxinhao.online"
```

如果这个命令能返回内容，说明 HTTPS 配置正常，只是域名解析问题。

### 步骤 3：检查本地 hosts 文件

```bash
cat /etc/hosts | grep tangxinhao
```

### 步骤 4：测试 HTTP（80 端口）

```bash
curl http://www.tangxinhao.online
```

或者：
```bash
curl http://118.190.106.159 -H "Host: www.tangxinhao.online"
```

## 解决方案

### 如果域名解析不正确

需要检查 DNS 配置，确保 `www.tangxinhao.online` 的 A 记录指向 `118.190.106.159`。

### 如果直接用 IP 可以访问

说明 HTTPS 配置正常，只是域名解析问题。可以：
1. 检查 DNS 配置
2. 或者暂时在本地 hosts 文件中添加映射（仅用于测试）
